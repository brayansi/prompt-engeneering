# Instruções para mensagens de commit

Ao gerar mensagens de commit:

1. **Formato**: Use Conventional Commits quando fizer sentido:
   - `tipo(escopo): descrição curta` (ex: `docs(nginx): adiciona guia de volumes`)
   - Tipos comuns: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

2. **Língua**: Escreva em português (descrição e corpo, se houver).

3. **Assunto**:
   - Uma linha, até ~72 caracteres.
   - Comece com verbo no imperativo (ex: "Adiciona", "Corrige", "Atualiza").
   - Não termine com ponto.

4. **Corpo** (opcional): Use quando precisar explicar o "porquê" ou o contexto; deixe uma linha em branco após o assunto.

5. **Consistência**: Mantenha o mesmo estilo dos últimos commits do repositório.
